# HaGeZi DNS Combo

Неофициальный инструмент для объединения списков [HaGeZi DNS Blocklists](https://github.com/hagezi/dns-blocklists). Не связан с автором HaGeZi и не является частью проекта hagezi/dns-blocklists.

Составной DNS-блоклист для повышения конфиденциальности и защищённости пользователя. Объединяет 13 списков доменов проекта [HaGeZi DNS Blocklists](https://github.com/hagezi/dns-blocklists) с последующей дедупликацией записей.

Предназначен для использования с DNSCrypt в [InviZible Pro](https://github.com/Gedsh/InviZible), [dnscrypt-proxy](https://github.com/DNSCrypt/dnscrypt-proxy) и других клиентах, поддерживающих импорт пользовательских блоклистов.

## О проекте

Блокировка осуществляется на этапе разрешения доменных имён: запросы к перечисленным доменам не получают корректного ответа DNS, что препятствует установлению соединений с соответствующими ресурсами.

Скрипт `Multi_HaGeZi.go` загружает исходные списки параллельно, объединяет их и исключает повторяющиеся домены. Итоговый файл `M_HaGeZi.txt` содержит только уникальные записи без пересечений между категориями.

## Состав репозитория


| Файл              | Назначение                                                           |
| ----------------- | -------------------------------------------------------------------- |
| `Multi_HaGeZi.go` | Скрипт сборки: загрузка исходных списков, объединение и дедупликация |
| `M_HaGeZi.txt`    | Итоговый блоклист (формируется при запуске скрипта)                  |


Файл `M_HaGeZi.txt` не включается в репозиторий: он генерируется локально и может достигать объёма около 40 МБ. Для распространения готового блоклиста рекомендуется публиковать его в разделе [Releases](https://docs.github.com/en/repositories/releasing-projects-on-github/about-releases).

## Формирование итогового списка



### Загрузка

Скрипт выполняет параллельную загрузку всех 13 исходных списков. Каждый список обрабатывается в отдельном потоке выполнения.

### Дедупликация

Все полученные записи проходят через механизм исключения повторов. Каждое доменное имя включается в итоговый файл один раз, независимо от того, в скольких исходных списках оно присутствовало.

В результате:

- список полностью уникален;
- пересечения доменов между категориями устранены;
- объём итогового файла оптимизирован за счёт отсутствия дубликатов.



### Формат выходного файла

Каждая строка файла `M_HaGeZi.txt` содержит одно доменное имя. Формат соответствует требованиям DNSCrypt для импорта блоклистов.

## Требования



### Для сборки блоклиста

- [Go](https://go.dev/dl/) версии 1.x или выше;
- активное подключение к сети Интернет;
- не менее 100 МБ свободного дискового пространства.



### Для использования блоклиста

- приложение с поддержкой DNSCrypt и импорта пользовательских блоклистов;
- достаточный объём памяти устройства для обработки списка (см. раздел [Особенности эксплуатации](#особенности-эксплуатации)).



## Сборка блоклиста



### Клонирование репозитория

```bash
git clone https://github.com/Mint1024100/hagezi-dns-combo.git
cd hagezi-dns-combo
```



### Запуск

```bash
go run Multi_HaGeZi.go
```

Либо предварительно скомпилируйте и запустите исполняемый файл:

```bash
go build -o Multi_HaGeZi Multi_HaGeZi.go
./Multi_HaGeZi
```



### Результат

По завершении работы скрипта в текущем каталоге создаётся файл `M_HaGeZi.txt`. Сообщения об ошибках сетевого доступа или записи на диск выводятся в консоль.

### Периодичность обновления

Исходные списки HaGeZi обновляются регулярно. Рекомендуется пересобирать блоклист не реже одного раза в 1–3 месяца либо при изменении условий использования.

## Подключение блоклиста



### InviZible Pro

1. Убедитесь, что модуль DNSCrypt активирован.
2. Перейдите в меню: **Меню → Настройки DNSCrypt**.
3. Выберите пункт **Импорт чёрного списка** (Import blacklist).
4. Укажите файл `M_HaGeZi.txt`.
5. Перезапустите модуль DNSCrypt при необходимости.

> **Примечание.** Функция импорта пользовательских блоклистов может быть недоступна в версии приложения, распространяемой через Google Play. Используйте сборку с [официального сайта](https://invizible.net), F-Droid  или [репозитория проекта](https://github.com/Gedsh/InviZible).



### Прочие приложения

При использовании иных клиентов DNSCrypt руководствуйтесь документацией приложения. Общее требование: файл должен содержать список доменных имён, по одному на строку, в кодировке UTF-8.

## Особенности эксплуатации



### Объём блоклиста

Итоговый файл содержит порядка 2,4 млн уникальных доменных записей и занимает около 40 МБ на диске.

InviZible Pro предупреждает, что импорт блоклистов объёмом более 5 МБ может повышать энергопотребление устройства. При использовании на мобильных устройствах учитывайте данное ограничение.

### Агрессивность фильтрации

Составной список включает категории с жёсткой политикой блокировки (Pro++, TIF, NSFW, Gambling и др.). Возможны следующие последствия:

- блокировка рекламных и аналитических доменов в приложениях и на сайтах;
- ограничение доступа к ресурсам азартных игр и контента для взрослых;
- нарушение работы отдельных функций приложений, использующих заблокированные домены для телеметрии или обновлений;
- ложные срабатывания на легитимные ресурсы в категориях Fake и Anti-Piracy.

При возникновении проблем с доступом к необходимому ресурсу используйте белый список (whitelist) в настройках DNSCrypt соответствующего приложения.

### Ограничения метода

DNS-блокировка:

- не шифрует содержимое сетевого трафика (для этого используются Tor, VPN и аналогичные средства);
- не блокирует обращения по прямому IP-адресу, минуя DNS;
- не заменяет комплексные меры информационной безопасности.

Рекомендуется использовать блоклист совместно с шифрованием DNS (DNSCrypt) и иными модулями защиты, предусмотренными используемым приложением.

## Устранение неполадок


| Симптом                         | Возможная причина                            | Рекомендуемое действие                                            |
| ------------------------------- | -------------------------------------------- | ----------------------------------------------------------------- |
| Ошибка загрузки URL при сборке  | Отсутствие доступа к GitHub                  | Проверьте подключение к Интернету и повторите запуск              |
| Файл не создаётся               | Недостаточно прав на запись                  | Запустите скрипт в каталоге с правами записи                      |
| Приложение не принимает файл    | Несовместимый формат или ограничение размера | Убедитесь в поддержке импорта; при необходимости разделите список |
| Сайт или приложение не работает | Домен попал в блоклист                       | Добавьте домен в белый список DNSCrypt                            |
| Повышенный расход батареи       | Большой объём блоклиста                      | Рассмотрите использование сокращённого набора списков             |
